## 1. 背景と目的

### 1.1 背景
SSI事業部では、未経験エンジニアがアカデミア生からスタートし、QAエンジニアを起点に各専門領域へ成長していくキャリアマップを定義している。各メンバーが「いま自分がどのレベルにいて、次に何を学ぶべきか」を把握するため、スキルマップ（スキル一覧）とスキルチェック表（5段階自己評価）をExcelで運用している。

現状は以下の課題がある。

- Excelをメンバー人数分コピーして配布・回収する運用で、集計と履歴管理に手間がかかる
- 達成率や到達レベルがひと目で把握しづらく、面談での活用がしにくい
- リーダー・ボードがチーム／全社のスキル状況を俯瞰する仕組みがない
- スキルマップが4トラック（アカデミア・テスト自動化・マネジメント・セキュリティ）分あり、拡張時の運用負荷が高い

### 1.2 目的
- メンバーが自分のスキルを直感的なUIで自己評価し、達成率・進捗・到達レベルを可視化できるようにする
- リーダーが自チームのスキル状況を、ボードが全社のスキル状況を把握できるようにする
- 自己評価の履歴を蓄積し、成長の推移と面談での活用を可能にする
- 将来的な学習ツールとのデータ連携を見据えた拡張可能な基盤を構築する

### 1.3 スコープ
- 本リリース（MVP）：Academia・テスト自動化・マネジメント・セキュリティの4コースを対象に、自己評価・ダッシュボード・チーム集計・管理画面を提供する
- v3.0追加：資格管理・キャリアパス・一括招待+GASメール送信・GCP Cloud Runホスティング移行
- 将来拡張：学習ツール連携

## 2. 利用者と役割（ロール）

| ロール | 主な利用者 | できること |
|---|---|---|
| メンバー（member） | SSI所属の全社員 | 自分のスキル自己評価、自分の達成率・進捗・到達レベルの確認、自チームメンバー名の閲覧 |
| リーダー（leader） | 各チームリーダー | 自チームメンバーのスキル状況・達成率の把握（メンバーと同じ画面）、面談での活用、提出履歴の削除 |
| ボード／管理者（board） | 経営層・管理者 | 全社員の得点一覧・進捗率・レベル別人数サマリーの俯瞰、マスタ管理、ユーザー管理、提出履歴の削除 |
| 退職者（retired） | 退職済み社員 | ログイン不可。データは管理者画面に残るがメンバー一覧には非表示 |

権限は上位互換とせず、明示的に付与する。リーダーは自チームのみ、ボードは全社を閲覧できる。メンバーは自分のデータ＋自チームメンバーの名前のみアクセスできる。複数チーム兼任リーダーは現時点で想定しない。

## 3. 機能要件

### 3.1 メンバー機能
- FR-M1：コース選択画面で4コース（Academia／テスト自動化／マネジメント／セキュリティ）から選択できる。各コースの概要・ゴールを表示する。複数コースを同時に進めてもよい旨を明記する
- FR-M2：各スキルについて5段階（1〜5）で自己評価を入力できる。回答は1問ずつ、16personalities風の直感的なUIで進む
  - 横並びの5つの丸（バブル）で選択する方式。左端「興味はある（1）」〜 右端「人に教えられる（5）」（左小→右大）
  - 丸のサイズは左が小さく右が大きい（左小→右大のグラデーション的サイズ変化）
  - 選択時にWidsleyブランドカラー（Sea Green→Cyanグラデーション）で塗りつぶし
  - 各問にスキル名、説明、カテゴリ／レベルのタグ、重み（難易度）を表示
  - 選択すると自動的に次の問へ進む（前の問に戻って変更も可）
  - JSTQB FL合格の質問のみ「合格」「まだ」の2択ボタンで表示
- FR-M3：全問回答しなくても提出可能（未回答は0点計算）。ブラウザを閉じたら未提出の回答は消える
- FR-M4：2回目以降の回答時、前回の回答が初期値としてプリセットされる（現在地マーキングのチェックシートとして運用）
- FR-M5：回答完了後、ダッシュボードで以下を確認できる
  - 総合達成率（リング表示）
  - カテゴリ別／レベル別の達成率
  - 到達レベルとレベルゲート判定
  - 次に伸ばすとよいスキルの提案（低スコア＋低難易度のものを優先）
  - 「詳細」ボタンから各問の回答内容一覧（スキル名・回答スコア・重み・加重スコア）
- FR-M6：提出ごとに履歴として保存される。2回目以降は前回差分との比較表示
- FR-M7：過去の提出の達成率の推移を折れ線グラフで確認できる（X軸：提出日、Y軸：達成率）
- FR-M8：自分のチームメンバーの名前を確認できる（レベル表示は後日検討）

### 3.2 リーダー機能
- FR-L1：自チームのメンバー一覧を表示できる
- FR-L2：メンバーごとの現在レベル・達成率・最終更新日を一覧で確認できる
- FR-L3：メンバー個別のダッシュボード・回答詳細をメンバーと同じ画面で確認できる
- FR-L4：面談に使えるよう、メンバーの強み・次に伸ばすスキルを参照できる
- FR-L5：メンバーの提出履歴を削除できる（コース単位・提出単位の選択削除をモーダルで行う）
- FR-L6：boardロールを兼務するリーダーは全チームを閲覧できる（チームセレクタードロップダウンで切替）

### 3.3 ボード／管理者機能
- FR-B1：全社員の得点（達成率）一覧を表示できる
- FR-B2：レベル別人数サマリーを表示できる
- FR-B3：コース別・チーム別に進捗率を集計して確認できる
- FR-B4：（管理）コース・レベル・スキル・重みのマスタをアプリの管理画面から管理できる
- FR-B5：（管理）ユーザーのロール割り当て・チーム振り分けを管理画面から行える
- FR-B6：（管理）チームの新規登録・名前編集・リーダー設定を管理画面から行える
- FR-B7：（管理）メンバーの提出履歴を削除できる
- FR-B8：（管理）四半期の回答催促通知を管理画面から送信できる（Slack連携）
- FR-B9：招待管理（→ 3.6 招待機能を参照）

### 3.4 資格管理機能
- FR-CERT1：資格表ページで報奨金対象資格の一覧を表示できる（報奨金対象のみ）
- FR-CERT2：キャリアパス推奨資格として、報奨金対象外を含む全資格を表示できる
- FR-CERT3：各資格に対して3ボタンのステータス選択（未取得／勉強中／取得済み）をモーダルで行える
- FR-CERT4：マイページで自分の資格取得状況を管理・確認できる

### 3.5 キャリアパス機能
- FR-CP1：職種選択画面で対象職種を選択できる
- FR-CP2：選択した職種のキャリアルート（ステップ）を表示できる
- FR-CP3：キャリアパスのスコープを登録し、マイページと連携できる
- FR-CP4：キャリアパスに紐づく推奨資格を表示し、資格管理と連携できる

### 3.6 招待機能
- FR-INV1：管理画面から最大10件の一括招待が可能。メール+ロール+チームを事前登録し、招待メール（Google Apps Script経由）を自動送信する
- FR-INV2：ユーザーが初回Googleログイン時に招待情報から自動でロール・チーム設定される

### 3.7 共通機能
- FR-C1：Googleアカウント（@widsley.com）でログインできる。それ以外のドメインはログイン不可
- FR-C2：ログインユーザーのロールに応じて表示・操作可能な範囲が制御される
- FR-C3：URLで社内共有でき、ブラウザ（PC・スマートフォン）で利用できる
- FR-C4：初回ログイン時にチーム未割当の場合、「管理者に連絡してください」と表示する。ただしチーム未割当でもコース回答は可能

### 3.8 Slack連携機能（後期実装可）
- FR-S1：管理画面でメンバーのSlack IDを連携設定できる
- FR-S2：メンバーが回答提出時、リーダー＋管理者に自動Slack通知が届く
- FR-S3：管理者が四半期の回答催促通知を送信できる。対象は当該期間に未回答の人
  - Academiaは卒業認定済みなら対象外
  - 各専門コースは毎回全員対象（全スキル5点＝100%の場合は通知対象外）

## 4. スコアリング・判定ロジック（要件）

### 4.1 加重スコアと達成率
- 各スキルに重み（weight）を設定する（確定済み、統合版スキルマップExcelの値が正）
- JSTQB FL合格は達成率計算から**除外**（合格/まだの2択で別軸管理）
- 加重スコア = Σ（自己評価値 × 重み）。未回答は0点として分母に含める
- 満点 = Σ（重み × 5）
- 達成率(%) = 加重スコア ÷ 満点 × 100（**切り捨て、小数第2位**まで）

### 4.2 レベルゲート判定
- 各レベルに属する全スキルが「評価**3以上**」かつ「全て回答済み」のとき、そのレベルを「到達」とする
- 評価2以下が1つでもあると、そのレベルでゲートが閉じる（次レベルへ進めない）
- 到達レベルは、下位レベルから順にゲートを満たす限り進む
- セキュリティコースはEntryなし、Associateからスタートの3段階

### 4.3 Academia卒業条件（必須）
- **JSTQB FL合格** AND **JSTQB以外の全スキル達成率80%超**
- 卒業時に「JSTQB FL取得済み」バッジを表示
- 卒業後も任意で再回答可能（ロックしない）。ただし四半期の回答催促通知は対象外

### 4.4 表示
- 画面では「レベル到達（ゲート判定）」と「達成率(%)」を必ず両方表示する
- 各コースの到達レベルを一覧表示：例）`JSTQB FL取得済み　自動化Associate　マネジメント-　セキュリティEntry`
- 未開始のコースは「-」で表示

## 5. 非機能要件

| 区分 | 要件 |
|---|---|
| 可用性 | 業務時間帯に安定稼働。無料枠インフラを基本とする |
| 性能 | 回答画面の操作レスポンスは体感即時。ダッシュボード表示は2秒以内を目安 |
| セキュリティ | 行レベルセキュリティでロール別アクセス制御。社外非公開（@widsley.comのみ認証） |
| プライバシー | 個人の自己評価は本人・自チームリーダー・ボードのみ閲覧可。SSN等の機微情報は扱わない |
| 対応端末 | PC・スマートフォンのモダンブラウザ（レスポンシブ対応） |
| 保守性 | スキルマスタはコードに直書きせず、管理画面から更新できる |
| 拡張性 | コース追加、学習ツール連携を見据えたデータモデルとする |
| バリデーション | 管理画面のみ（チーム名の空文字・重複、スキルマスタの重み・名前チェック、ロール変更確認ダイアログ） |
| テキスト表示 | スキル説明・コースゴール等の長文テキストは左寄せ（left-aligned）で可読性を確保 |
| ブランディング | favicon=Widsleyロゴ、ページタイトル=Widsley SkillCheck |

## 6. 技術構成（決定事項）

| 層 | 採用技術 | 理由 |
|---|---|---|
| フロントエンド | React + TypeScript | プロトタイプからの継続、開発資産との親和性 |
| ホスティング | GCP Cloud Run（asia-northeast1） | コンテナベースで安定運用、東京リージョン |
| バックエンド／DB | Supabase（PostgreSQL、東京リージョン） | DB・認証・自動API・行レベルセキュリティが無料枠で完結 |
| 認証 | Supabase Auth（Google / @widsley.com） | 既存Google Workspaceと整合 |
| 通知 | Slack Bot（後期実装可） | 回答提出通知・四半期催促通知 |
| ソース管理 | GitHub | 既存運用 |

コスト方針：原則すべて無料枠で構築する。サーバー費用が発生する場合（Supabase Pro 約$25/月 等）は事前に申請する。

インフラ構築順序：
1. Supabaseプロジェクトを個人アカウントで新規作成（東京リージョン）
2. 実装後に社長承認を得てから会社アカウントに移行
3. 現時点は本番環境のみ。リリース後に新機能追加時にステージング環境を追加検討

## 7. 前提・制約・リスク

- Supabase／GCP／Google認証のアカウント作成・連携設定は管理者（ユーザー本人）の操作が必要
- GCP Cloud RunへのデプロイおよびカスタムドメインのDNS設定はユーザー本人が実施
- Slack Bot作成・トークン発行はユーザー本人が実施
- スキルマスタ（重み・レベル割当）は事業部の合意済み定義に従う（統合版スキルマップExcel）。変更時は履歴データとの互換に注意（スナップショット方式で対応）
- UI決定事項：メンバー画面では重み(weight)を「難易度(★)」として表示し、加重スコアは非表示。リーダー／ボード画面では重み・加重スコアを両方表示する。面談メモ用のtextareaをリーダー画面に設置（localStorageに保存）。回答詳細画面ではスキル説明（description）を表示する
- Supabase無料枠は7日間非アクティブでプロジェクト一時停止あり（100人利用なら問題なし）
- バックアップ：月1回手動pg_dumpで対応。日次自動バックアップはあるがリストアはPro以上
- 障害時の対応はオペレーション推進室（現時点ではユーザー本人）

## 8. 受け入れ基準（MVP）

- [ ] メンバーがGoogleログインし、4コースの自己評価を完了・提出できる
- [ ] 全問回答しなくても提出でき、2回目以降は前回回答が引き継がれる
- [ ] 回答完了後、達成率・到達レベル・ゲート判定がダッシュボードに表示される
- [ ] 詳細ボタンから各問の回答内容一覧を確認できる
- [ ] JSTQB FL合格が2択で別軸管理され、Academia卒業条件が正しく判定される
- [ ] 提出履歴が保存され、前回差分比較・推移グラフが表示される
- [ ] リーダーが自チームのメンバー一覧と達成率を確認できる
- [ ] ボードが全社の得点一覧とレベル別人数サマリーを確認できる
- [ ] 管理画面からマスタ管理・ユーザー管理・チーム管理ができる
- [ ] ロール別のアクセス制御が機能する（他人のデータが見えない）
- [ ] カスタムドメインで社内に共有でき、スマートフォンでも操作できる
- [ ] 初回ログイン時にチーム未割当でも回答可能で、管理者連絡の案内が出る

## 9. 運用方針

### 9.1 評価サイクル
- **四半期ごと**に回答を実施
- 管理者が期限を設定し、未回答者にSlack通知で催促
- Academiaは卒業認定済みなら対象外
- 各専門コースは毎回全員対象（全スキル5点＝100%の場合は通知対象外のみ）

### 9.2 スキルマスタ更新
- 管理者のみが管理画面から更新
- 頻度は年2回程度
